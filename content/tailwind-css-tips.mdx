# 10 Tailwind CSS Tips for Better Development

Tailwind CSS has revolutionized how we write styles. Here are 10 essential tips to boost your productivity and write better Tailwind code.

## 1. Use the `@apply` Directive Wisely

Extract repeated utility patterns into custom classes:

```css
@layer components {
  .btn-primary {
    @apply px-4 py-2 bg-blue-500 text-white rounded-lg hover:bg-blue-600 transition-colors;
  }
}
```

> **Note:** Don't overuse `@apply`. The utility-first approach is powerful on its own!

## 2. Leverage the JIT Compiler

Tailwind's Just-In-Time compiler enables arbitrary values:

```html
<div class="w-[137px] h-[87px] top-[-113px]">Custom sizes and positions!</div>
```

## 3. Use Responsive Design Modifiers

Tailwind makes responsive design simple:

```html
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <!-- Responsive grid -->
</div>
```

Available breakpoints:

- `sm:` - 640px
- `md:` - 768px
- `lg:` - 1024px
- `xl:` - 1280px
- `2xl:` - 1536px

## 4. Master Dark Mode

Implement dark mode with the `dark:` variant:

```html
<div class="bg-white dark:bg-gray-900 text-black dark:text-white">
  Automatic dark mode support!
</div>
```

Configure in `tailwind.config.js`:

```javascript
module.exports = {
  darkMode: "class", // or 'media'
};
```

## 5. Create Custom Utilities

Extend Tailwind with your own utilities:

```javascript
module.exports = {
  theme: {
    extend: {
      spacing: {
        128: "32rem",
        144: "36rem",
      },
      colors: {
        "brand-blue": "#1fb6ff",
      },
    },
  },
};
```

## 6. Use Group and Peer Modifiers

Style children based on parent state:

```html
<!-- Group -->
<div class="group">
  <img class="group-hover:opacity-75" src="..." alt="..." />
</div>

<!-- Peer -->
<input type="checkbox" class="peer" />
<label class="peer-checked:text-blue-500">Checked!</label>
```

## 7. Optimize with Purging

Tailwind automatically purges unused styles in production:

```javascript
module.exports = {
  content: [
    "./app/**/*.{js,ts,jsx,tsx,mdx}",
    "./components/**/*.{js,ts,jsx,tsx,mdx}",
  ],
};
```

## 8. Use Container Queries

Modern container queries are now supported:

```html
<div class="@container">
  <div class="@lg:grid-cols-2">
    <!-- Responds to container size -->
  </div>
</div>
```

## 9. Master Arbitrary Variants

Apply styles based on any selector:

```html
<div class="[&>*]:p-4 [&_img]:rounded-lg">
  <!-- Styles all children and nested images -->
</div>
```

## 10. Use the Tailwind CSS IntelliSense

Install the official VS Code extension for:

- Autocomplete
- Linting
- Hover previews
- Syntax highlighting

```bash
code --install-extension bradlc.vscode-tailwindcss
```

## Bonus: Useful Plugins

Enhance Tailwind with official plugins:

```javascript
module.exports = {
  plugins: [
    require("@tailwindcss/forms"),
    require("@tailwindcss/typography"),
    require("@tailwindcss/aspect-ratio"),
    require("@tailwindcss/container-queries"),
  ],
};
```

## Pro Tips

### Combine Utilities Smartly

```html
<!-- Instead of repeating styles -->
<button class="px-4 py-2 bg-blue-500 text-white rounded">Button 1</button>
<button class="px-4 py-2 bg-blue-500 text-white rounded">Button 2</button>

<!-- Extract to a component -->
<button>Button 1</button>
<button>Button 2</button>
```

### Use CSS Variables for Theming

```css
:root {
  --color-primary: 59 130 246;
}

.dark {
  --color-primary: 37 99 235;
}
```

```html
<div class="bg-[rgb(var(--color-primary))]">Theme-aware background</div>
```
